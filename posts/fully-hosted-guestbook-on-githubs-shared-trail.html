Fully Hosted: A Guestbook on GitHub’s Shared Trail

By Cris, April 28, 2025

Along the web’s shared trail, where creators wander like travelers seeking connection, I envisioned a simple shelter—a guestbook app hosted entirely on GitHub, free for all, with no servers to anchor the journey. Fully Hosted took shape as a proof of concept, using GitHub Pages to paint a static front-end and GitHub Actions to carve a living database, an SQLite haven where messages could be written and read as naturally as notes left in a forest clearing. It was a quiet experiment, a space where anyone could leave their mark, hinting at grander dreams—like a collaborative code editor blooming from the same earth.

The trail began with GitHub Pages, a canvas for the guestbook’s HTML form, where users sign their names and messages, served as static whispers on the wind. Submitting an entry triggers a GitHub Action, a steady hand that writes to an SQLite database using sanitized inputs, then commits the changes back to the repository with a private SSH token, while a public token safely initiates the workflow. On the client side, `sql.js` reads the database directly in the browser, displaying entries like a trail of footprints, a seamless flow from write to read without a server’s shadow. This hack opened a path to endless possibilities—imagine a GitHub Pages code editor where anyone could collaboratively shape a repo’s code, a shared campfire of creation.

# GitHub Action workflow for Fully Hosted guestbook
name: Update Guestbook Database
on:
  workflow_dispatch:
    inputs:
      name:
        required: true
      message:
        required: true
jobs:
  update-db:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Sanitize inputs
        run: |
          node -e "const sanitizeHtml = require('sanitize-html');
            const name = process.env.NAME;
            const message = process.env.MESSAGE;
            console.log('SANITIZED_NAME=' + sanitizeHtml(name, { allowedTags: [] }));
            console.log('SANITIZED_MESSAGE=' + sanitizeHtml(message, { allowedTags: [] }));"
        env:
          NAME: ${{ inputs.name }}
          MESSAGE: ${{ inputs.message }}
      - name: Insert guestbook entry
        run: |
          sqlite3 data.db "CREATE TABLE IF NOT EXISTS guestbook (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT, message TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);"
          sqlite3 data.db "INSERT INTO guestbook (name, message) VALUES ('$SANITIZED_NAME', '$SANITIZED_MESSAGE');"

The workflow above traces Fully Hosted’s gentle rhythm—an Action that sanitizes and writes guestbook entries to an SQLite database, a living record of shared moments committed back to the repo. The innovation lay in its simplicity: GitHub Actions became a serverless database, writing data as naturally as carving words into stone, while Pages and `sql.js` brought the entries to life in the browser, a seamless read without server-side overhead. A dual-token system—public for triggering, private for committing—kept the shelter secure, proving that GitHub’s tools could host dynamic apps, opening trails to dreams like collaborative code editing, as free and open as the sky above a traveler’s path.

In crafting Fully Hosted, I discovered the beauty of shared trails—how GitHub’s infrastructure can become a canvas for connection, its Actions and Pages a shelter for creativity. It stands as a reminder that even in the web’s vast expanse, we can build spaces of simplicity, as natural as the earth’s own offerings. Like a rider pausing to leave a note in a clearing, or a skydiver glimpsing a shared path below, Fully Hosted is a testament to the power of community, a quiet proof that whispers of code can grow into campfires of collaboration, carried forward on the web’s endless journey.
